    react

    When you see something, your should think more of it, the more you think, the better you get. You should know what the advantage of Java is and when met with some code beautiful, think about its implementation and try implement in other languages.
    React thinks of UIs as simple state machines, and rendering those states. Try to keep as many of your components stateless as possible. State should contain data that a component’s event handlers may change to trigger a UI update. when building a stateful component, think about the minimal possible representation of its state and only store those properties in this state. A component can’t mutate its props, they are always consistent with what its owner sets them to.
